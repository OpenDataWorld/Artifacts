# AnyDB Foundation Principle: The Domain Makes Things Real

Status: Normative Foundation Principle

## Principle

> The domain makes a thing real by defining the context in which it has meaning, dimensions, state, rules, measurements, boundaries, and consequences.

A thing outside a domain is only an undefined possibility.

When a domain accepts, interprets, measures, and governs that thing, it becomes real within that domain.

## Core Rule

> Reality is not inherent in an isolated object. Reality emerges when a domain gives the object context, boundaries, dimensions, and valid state.

## Canonical Formula

```text
Thing + Domain + Time + Dimensions + Evidence = Real
```

And:

```text
AnythingKube + Domain Context = Governed Reality
```

## Domain as the Reality Boundary

A domain defines:

- what the thing is
- which schema applies
- which dimensions matter
- how it is measured
- which states are valid
- which transitions are allowed
- which laws and policies apply
- who may observe or act
- which evidence is sufficient
- when the thing exists
- where the thing may operate
- how it may enter or leave the domain

## Possibility to Reality

```text
Undefined Thing
      ↓
Domain Admission
      ↓
Identity Resolution
      ↓
Schema and Dimension Binding
      ↓
Policy and Law Evaluation
      ↓
State Acceptance
      ↓
```

## Nothing Is Universally Canonical

No object is universally canonical in isolation.

A version may become accepted, authoritative, or real only within a declared domain, time, location, observer, purpose, and contract.

Recommended terms:

```text
locally authoritative
domain-authoritative
accepted version
reconciled version
observed version
current governed version
```

## One World, Many Realities of Context

There is one world, but many domains may make different fractions of the same thing real.

```text
One Thing
   |
   +-- Finance Reality
   +-- Legal Reality
   +-- Operational Reality
   +-- Cultural Reality
   +-- Research Reality
```

These are not unrelated worlds.

They are contextual realities of one shared thing.

## Domain Admission

A domain makes a thing real only after admission.

Admission may require:

- identity
- passport and visa
- schema compatibility
- policy conformance
- legal eligibility
- evidence
- time validity
- location validity
- trust threshold
- no unresolved critical gaps

## Domain-Specific Dimensions

The domain chooses the dimensions that matter.

Examples:

- finance: value, currency, exposure, settlement
- healthcare: diagnosis, severity, consent, treatment window
- logistics: location, custody, condition, delivery state
- research: hypothesis, evidence, confidence, reproducibility
- culture: language, meaning, tradition, sensitivity

## Domain and Measurement

A thing becomes operationally real when the domain can measure it.

```text
Measurement
=
Value
+ Unit
+ Domain Meaning
+ Time
+ Location
+ Observer
+ Evidence
```

## Domain and Time

Time determines when the domain realization is valid.

The same thing may be:

- future-real
- currently real
- historically real
- expired
- superseded
- simulated

The temporal mode must be explicit.

## Domain and Location

Location determines where the domain realization applies.

A thing may be valid in one jurisdiction, edge, cluster, or cultural region and invalid in another.

## Domain and AnyAgent

AnyAgent may act only in a domain that has admitted and contextualized it.

The domain defines:

- objective semantics
- allowed tools
- evidence requirements
- autonomy limits
- intervention authority
- stop conditions

## Domain and Memory

Memory is a map.

The domain is the legend that makes the map readable.

Without domain context, memory nodes may exist but remain ambiguous.

## Domain and Context Cloud

The Context Cloud delivers domain-realized context to each AnythingKube and AnyAgent.

```text
Memory Map
   ↓
Domain Interpretation
   ↓
Purpose-Bound Context
   ↓
AnythingKube / AnyAgent
```

## Domain Exit

When a thing exits a domain, its realization in that domain becomes historical.

The thing may enter another domain through a new forward admission.

```text
Domain A Reality
      ↓ exit
Journey
      ↓ entry
Domain B Reality
```

Identity continues. Domain reality changes.

## Federation

Domains federate by exchanging governed realizations and mappings.

A federation must preserve:

- source domain
- target domain
- semantic mapping
- time
- evidence
- authority
- declared loss
- reconciliation

## Reconciliation

Reconciliation evaluates whether a domain realization still matches:

- observed state
- valid policy
- current law
- evidence
- time
- location
- contract

A thing remains real in the domain only while those conditions continue to hold.

## Platform Invariants

1. A thing is not real in a domain before admission.
2. Domain defines meaning, dimensions, state, and measurement.
3. Domain reality is time- and location-bound.
4. No domain silently claims universal authority.
5. One thing may have many contextual realizations.
6. Domain mappings preserve provenance and declared semantic loss.
7. AnyAgent operates only inside admitted domain context.
8. Domain exit ends the current realization but not identity history.
9. Re-entry is a new forward realization.
10. Reconciliation continuously verifies domain reality.

## Final Statement

> The domain makes things real. It turns an undefined possibility into a governed existence by assigning meaning, dimensions, state, law, evidence, time, and consequence. Nothing is universally canonical; everything becomes real within context.
